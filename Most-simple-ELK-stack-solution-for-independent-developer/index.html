Most simple elk stack solution for independent developer

I’m a IT consultant and independent developer. There are so many different projects I have developed for such a long time and most of these projects don’t have lot traffic. So I think this solution is the most simple.

TLDR: All project got a filebeat container to watch log files and send to a SINGLE ELK sever(Elasticsearch/Logstash/Kibana all in one).

ELK server

  1. Install docker and docker compose. Offical docs.
  2. Clone git repo. All docker files is in elk_stack/elk.

  1. Open income 5044 port for Logstash. Highly recommend to open specific ips.

  2. Increase vm.max_map_count.

  1. Most simple authorization, use Nginx to redirect request from 80 to 5601(Kibana). Set username and password in Nginx for authorization, so that you don’t need to deal with X-pack.
  1. Generate pass file with htpasswd. The site_pass file will created in current path.
  1. Move site_pass to /etc/nginx/site_pass
  1. RUN~
  1. All things done. Kibana will be avaiable in http://your.elk.domain.

Project server(client)

Using filebeat to watch log files, all logs with be delivered to the ELK server.

  1. Install docker and docker compose. Offical docs.
  2. Clone git repo. All docker files is in elk_stack/filebeat.

  1. Edit filebeat.yml. Change host and port of target elk server, default port is 5044. Change tag of inputs section. It’s used to distinct from other project’s log. My name rule is hostname+project_name.

  2. Config log path of host to watch in docker-compose.yml, add volume like below, first one is log path of host.

  1. start docker-compose, and all thing done.
